# SOURCE_STANDARDS
- version: 1.0
- status: active
- last_updated: 2026-04-28

## Preferred Source Order

1. official docs
2. government or regulatory sources
3. manufacturer documentation
4. primary sources
5. reputable industry organizations
6. case studies
7. credible trade publications
8. high-quality expert blogs or articles
9. forums or social proof as weak support only

## Allowed Public Source Types

- official websites
- public laws, codes, records, and guidance
- manufacturer docs
- public API docs
- public help centers
- public case studies
- public trade articles
- public videos, posts, and forums when clearly labeled as weaker support

## Not Allowed

- private email
- private Dropbox
- private CompanyCam projects
- private Jobber or customer records
- paid or private databases unless Chris approves
- login-only data
- API keys, tokens, `.env`, or credentials

## Traceability Rule

If information may later be used in public content, emails, articles, website copy, or client-facing material, the source must be traceable and citation-ready.

Use `source_url_or_path` for either a public URL or an approved internal file path.

## Required Trace Fields

- `source_type`
- `source_name`
- `source_url_or_path`
- `date_accessed`
- `confidence_level`
- `citation_needed`
- `citation_available`
- `public_source`
- `private_internal_source`
- `notes_for_citation`

Clawdia may write Goonie memory or knowledge only when:

- Chris directly asks her to save it
