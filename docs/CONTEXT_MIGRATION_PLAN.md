# CONTEXT_MIGRATION_PLAN.md
> Phase-1 plan for moving important project knowledge into repo-based documentation.

## Purpose

A large amount of project context currently exists in chats, notes, and prior working sessions.
This document defines the plan for moving the important parts into durable repo-based documents.

## Migration Priorities

1. Agent system rules and templates
2. Aquatrace app architecture notes relevant to agent work
3. Known problem areas and recurring workflow issues
4. Approval and escalation logic
5. Reusable task/request patterns

## Migration Rules

- move only high-value repeat-use knowledge first
- avoid dumping raw chat transcripts into the repo
- rewrite context into practical documents
- prefer concise documents over long chat-history copies
- keep business logic separate from avatar or presentation ideas

## Phase 1 Goal

Create a documentation foundation so future work depends less on chat memory and more on stable repo files.
