# HANDOFF_WORKFLOW.md
> Phase-1 handoff workflow between human operator, Agent Architect, and future child agents.

## Purpose

This document defines the basic handoff flow for requesting, defining, reviewing, and tracking new agents.

## Phase-1 Workflow

1. Human identifies a need for a new agent
2. Human fills out docs/AGENT_REQUEST_TEMPLATE.md
3. Agent Architect uses that request to define a spec
4. Human reviews the draft spec
5. If approved, the new agent spec is added to the docs set
6. docs/AGENT_REGISTRY.md is updated with the new agent entry
7. Future implementation planning can begin after approval

## Phase-1 Notes

- This is a documentation-first workflow
- Human review remains required
- No live deployment or orchestration is assumed in phase 1
- Child agents should remain narrow in scope
