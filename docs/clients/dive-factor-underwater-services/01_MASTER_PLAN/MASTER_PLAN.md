## Immediate milestone targets

This milestone should produce an internal lane that is usable without creating public risk.

Required outputs:

1. isolated lane confirmed and organized
2. service buckets split into likely-ready vs. hold-for-review
3. Hartwell and Keowee research queue prioritized first
4. source-backed research standard defined
5. pricing framework documented without turning into a public rate card
6. B2B lead-list schema defined for Google Sheets
7. B2C, website, SEO, intake, and sales requirements documented at planning level
8. proof log updated with assumptions, blockers, and next steps

## Core workstreams

### 1. Business direction

- define what Dive Factor is
- define what Dive Factor is not
- decide launch geography
- decide first offer and first buyer types

### 2. Service strategy

- package lake-based inspection services first
- separate visual documentation from regulated or high-risk claims
- stage recurring plans only after operating model review
- use the Below-Waterline Inspection as the anchor offer for both B2C and partner conversations
- keep coastal work as a later branch unless compliance and operational review support it

### 3. Pricing framework

- build internal pricing logic
- identify travel, size, urgency, and documentation multipliers
- create a quote path before public pricing is finalized
- define minimum-scope assumptions behind the $295+ starting point
- separate quoting logic for homeowner, boat-only, dock-only, and partner scopes

### 4. Safety and compliance

- define what may be planned now
- define what must be held for review
- require proof before public claims about compliance, certification, or contractor status

### 5. Marketing and sales

- map B2B and B2C targets
- define website structure
- define keyword hypotheses
- prepare intake and scripts
- specify approval gates before any publish or outreach work begins

### 6. Build queue

- list future Codex tasks in the right order
- keep strategy before site build
- keep site build before tool automation

## First milestone exit criteria

The first milestone is complete when:

1. The client lane is fully scaffolded.
2. The first-offer strategy is documented.
3. Safety-sensitive offers are clearly marked.
4. Research queue exists with source requirements.
5. Pricing framework exists for internal use.
6. Intake, lead list, and script requirements exist.
7. Codex build queue is sequenced for Clawdia handoff.
