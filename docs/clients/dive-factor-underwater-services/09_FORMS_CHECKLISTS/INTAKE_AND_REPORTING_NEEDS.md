# Dive Factor Underwater Services - Intake and Reporting Needs

## Purpose

This file defines the milestone-one intake, checklist, and reporting requirements.

## Intake needs

### B2C request intake

- name
- phone
- email
- service area
- property or marina name
- asset type
- asset size or scope
- reason for request
- urgency
- preferred timing
- media upload if available
- permission or access notes
- requested documentation depth

### B2B inquiry intake

- business name
- category
- website
- market
- contact name
- contact role
- phone
- email
- fleet, dock, marina, or property count
- recurring-need description
- geography served
- vendor or COI requirements if known

## Checklist needs

### Pre-job checklist

- authorization confirmed
- location confirmed
- scope confirmed
- hazards discussed
- weather and water conditions checked
- access instructions confirmed
- documentation expectations confirmed
- exclusions acknowledged
- go/no-go safety screen completed

### Post-job checklist

- photo and video captured
- files organized
- written summary drafted
- limitations stated
- next action stated
- proof log entry created

## Reporting needs

- standard photo/video organization method
- standard written summary structure
- limitation/disclaimer block
- referral/recommendation block
- internal proof retention method
