# Server App

This is the Node/Express API host for NexTeam Studio. It owns the runtime API surface for health checks, version receipts, media proxying, approval queue endpoints, and module routers as each wave lands. It exposes the authenticated Nexi Job Desk, M2 native CRM client/import/quote/portal/invoice/Stripe routes, M3 native scheduling routes, and M4 native Field Docs upload, search, checklist, report record, and PDF routes.

It connects browser/mobile clients to the shared packages: `@nexteam/core` for contracts and zod validation, `@nexteam/providers` for Jobber, CompanyCam, and native CRM rails, `@nexteam/industry-packs` for catalog data, and `@nexteam/nexi` for assistant/gateway flows. Keep provider credentials in environment variables only. Scheduling uses `GOOGLE_MAPS_API_KEY` for Distance Matrix drive time when present, and falls back to a deterministic heuristic when absent.

When something breaks, start with `src/server.ts` for route wiring, `src/health.ts` for rail health failures, `src/firebase.ts` for Admin SDK setup, and the module folder under `src/` that owns the failing endpoint. For Nexi auth/chat issues, check `src/nexi/nexiRoutes.ts`, `src/nexi/nexiService.ts`, and `../web/src/main.tsx`. For scheduling, start with `src/scheduling/routes.ts` for API behavior, `src/scheduling/nexiTools.ts` for assistant tool calls, `src/scheduling/schedulingEngine.ts` for conflict/drive-time scoring, `src/scheduling/repository.ts` for native visit storage, and `src/scheduling/notifications.ts` for ApprovalQueue message shape. Booking, reminder, and on-my-way notifications must remain parked in ApprovalQueue; there is no direct send path in this module. For CRM quote or invoice issues, check `src/crm/routes.ts`, `src/crm/quoteBuilder.ts`, `src/crm/nativeRepository.ts`, `src/crm/quotePdf.ts`, and `src/crm/stripe.ts` in that order. For Field Docs, start with `src/fielddocs/routes.ts`, then check `uploadService.ts`, `mediaRepository.ts`, `photoSearch.ts`, `checklists.ts`, `reportService.ts`, and `visionPipeline.ts`. Outbound report delivery is intentionally not part of M4; live vision only runs when `FIELD_DOCS_VISION_ENABLED=true` and receipt runs must record spend.
